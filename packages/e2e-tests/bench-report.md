# Differential benchmark — Hasura (recorded baseline) vs envio serve

Baseline recorded 2026-07-02T11:10:19.951Z. 588 cases; per-case budget 1500ms, 3-30 iterations, warmup 2; case concurrency 1; sweep 415s.

**Release build.** Previous sweeps in this file were against a debug build
(`cargo build --lib`, the default dev-loop path in
`packages/envio/src/Core.res`). This sweep is against `cargo build --release
--lib` — geomean speedup **x4.81** vs the prior debug sweep's **x3.85**, peak
envio RSS **533MB** vs debug's **931MB**. Full differential parity
(release-and-debug both 596/596 on the default phase, 19/19 on limited)
confirmed unchanged before benchmarking. `em-deep-nesting-40-levels` (a
40-level relationship-nesting stress case, not representative of normal
queries) showed high run-to-run variance across repeated sweeps on this
machine (1.6ms-35.7ms), independent of build type or machine load at sweep
time; treat its single-run number with that caveat. `om-order-object-rel-nested-aggregate`
is a single ~30s pathological query (3 samples) where both builds are
within noise of the baseline.

**RSS load smoke test.** Ran `pnpm soak:differential -- --duration 45s
--concurrency 24` against this same release binary as a smoke check of the
soak harness itself (see `soak-report.md` for the full per-window table).
0 errors across 614 requests; p99 drift x1.18 (well under the x2 threshold).
RSS climbed from a 533MB cold start to a 1381MB "stabilized" baseline
(measured at 8-16s, before the pool's 24 connections had all been used at
least once) up to a 1906MB peak around 32-40s, then back down to 1728MB by
the end — the tool's >20%-from-early-baseline check flagged this as a FAIL,
but the shape (peak reached, then declining) reads as pool/statement-cache
warm-up settling within the run rather than sustained growth. A 45s window
is too short to tell warm-up apart from a real leak with confidence; **the
2h acceptance run this goal calls for (`pnpm soak:differential -- --duration
2h --concurrency 48 --spawn`) was not run in this session** (impractical
within an interactive session) and should be run before relying on this as
a production-readiness signal.
